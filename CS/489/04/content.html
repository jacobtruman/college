Problem 1

Write a program that creates, for a specified chromosome, a file named, for example for chromosome 01, 'chr01.exon-intron' and a similar file named 'chr01.intron-exon', with the different chromosome numbers and with the .exon-intron file containing the exon-intron boundaries and the .intron-exon file containing the intron-exon boundaries for each gene and corresponding introns, if any.

An example of input records and output records for a single gene in chromosome 08 are shown below.

The intron-exon boundary file records should be of the following 

   >LOC_Os08g17940|11978.t01664.m05913|intron_1-exon_2 unspliced-genomic hypothetical protein
   TCGGCTAGCAAGCAGGTCCATTGGGATGAA
   >LOC_Os08g17940|11978.t01664.m05913|intron_2-exon_3 unspliced-genomic hypothetical protein
   CCCCCTCCTCGCAAGAGTAACAGTGGAAGG
   >LOC_Os08g17940|11978.t01664.m05913|intron_3-exon_4 unspliced-genomic hypothetical protein
   ATCTCGTTTTGGCAGAAAAAGAAGACTGCT
   >LOC_Os08g17940|11978.t01664.m05913|intron_4-exon_5 unspliced-genomic hypothetical protein
   ATTTCTCTTTTCCAGACCTTAGCCGATCTA

A header identifier should have the information consisting of '>LOC_' followed by the species ID, the gene ID, the transcription unit number from the .seq header, a version number if there, the model number from the .intron header, the fact that it is the exon-intron (or intron-exon) boundary, and finally, the commentary, which should be the same for the .seq file and for the intron file..

The single body line contains the subsequence consisting of 10 bases before and 10 bases after the boundary for each record in the .seq file and the corresponding records in the .intron files (with NO helpful space in between, as shown in the earlier discussion but is not in the sample input/output.)

Similarly for the intron-exon boundaries.

The header line should have the following format: 

   >LOC_Os08g17940|11978.t01664.m05913|exon_1-intron_1 unspliced-genomic hypothetical protein
  ">LOC_Os" + "<chr #>" + "g" + "<gene number>" + "|" + "" + "." + "t" + "<TU #>
  + "." + "m" + "<model #>" + "|" + "intron_" + "<intron #>" + "-exon_" + "<exon #>"
  + " " + "<commentary>"

The command line for this program should have the usage: 

   Usage: exonIntron <.seq file name> <.intron file name> <chromosome number (beginning with 'chr')>
For example: 

	exonIntron chr01.dir/chr01.seq chr01.dir/chr01.intron chr01

The input files may not be in the current directory. The output file should be in the current directory and it's name can be figured from the chromosome number on the command line.
